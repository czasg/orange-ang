启动 Nginx 服务后，它内部会启动一个主进程和多个子进程，其中：

- 主进程：负责启动、停止、管理子进程，并接收外部信号，包括重载配置、服务停止等。
- 子进程：处理连接与请求，相互之间独立，某一个子进程异常不会影响其他子进程服务。

## 快速开始

在本章节中，我们将使用 nginx 默认配置启动服务，你可以暂时不用关心 nginx 默认配置是什么/有什么用~

### 1. 启动服务

```bash
$ docker run -it --rm -p 80:80 nginx
```

在上述指令中，我们启动了 nginx 服务，并对外暴露了 80 端口，因此我们可以通过访问本地 80 端口来验证 nginx 服务可用性。

### 2. 验证服务

通过浏览器访问 `http://localhost/`，如果可以看到 **Welcome to ngixn** 页面，那就说明 nginx 服务已经正常跑起来了。

## 自定义配置

在本章节中，我们将自定义 nginx 服务配置，并实现：

- 访问 `/index` 时返回自定义静态文件 index.html
- 访问 `/hello/{name}` 时返回 hello XXX

那如何理解 nginx 配置？它又有什么作用？

nginx 配置定义了请求的匹配规则，以及匹配之后的处理逻辑，因此配置文件是 nginx 开发过程中重点需要关注的。让我们先来看下 nginx 的默认规则配置文件。

```yaml title="/etc/nginx/conf.d/default.conf"
server {
    listen       80;
    server_name  localhost;

    #access_log  /var/log/nginx/host.access.log  main;

    location / {
        root   /usr/share/nginx/html;
        index  index.html index.htm;
    }

    #error_page  404              /404.html;

    # redirect server error pages to the static page /50x.html
    #
    error_page   500 502 503 504  /50x.html;
    location = /50x.html {
        root   /usr/share/nginx/html;
    }

    # proxy the PHP scripts to Apache listening on 127.0.0.1:80
    #
    #location ~ \.php$ {
    #    proxy_pass   http://127.0.0.1;
    #}

    # pass the PHP scripts to FastCGI server listening on 127.0.0.1:9000
    #
    #location ~ \.php$ {
    #    root           html;
    #    fastcgi_pass   127.0.0.1:9000;
    #    fastcgi_index  index.php;
    #    fastcgi_param  SCRIPT_FILENAME  /scripts$fastcgi_script_name;
    #    include        fastcgi_params;
    #}

    # deny access to .htaccess files, if Apache's document root
    # concurs with nginx's one
    #
    #location ~ /\.ht {
    #    deny  all;
    #}
}
```

在上述配置文件中，`#`后的内容表示注释内容，此外我们还可以看到`location`关键字，它就是用于定义请求的匹配规则，以及匹配之后的处理逻辑。

至于其他关键字比如`listen`、`server_name`、`error_page`等，我们可以暂时不用关注。

### 1. 准备文件

为了实现目标需求，我们需要准备以下两个文件：

- **default.conf**：表示 nginx 配置文件
- **index.html**：表示静态页面文件

文件内容分别为：

```yaml
server {
    listen       80;
    server_name  _;

    # 匹配以 /index 开头的请求
    location /index {
        # 指定根目录
        alias  /workspace;
        # 指定默认返回的文件
        index  index.html;
    }

    # 使用正则表达式捕获 URL 中的 {name} 参数
    location ~ /hello/(?<name>\w+) {
        # 设置响应头
        add_header Content-Type text/plain;
        # 返回响应
        return 200 "Hello $name";
    }
}
```

### 2. 启动服务

```bash
$ docker run -it --rm -p 80:80 \
    -v /workspace/default.conf:/etc/nginx/conf.d/default.conf \
    -v /workspace/index.html:/workspace/index.html \
    nginx
```

### 3. 验证服务

通过浏览器访问 `http://localhost/index`，如果可以看到 **Welcome to ngixn** 页面，那就说明 `location /index` 规则已经生效。
