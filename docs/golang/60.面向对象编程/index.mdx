在计算机编程中，面向对象编程是一种软件设计风格，通过将程序中的实体和现实世界中的对象关联起来，从而提高程序的可读性和可理解性。面向对象编程的主要优点之一是它可以帮助解耦模块对象，并提升代码的可复用性。

但可惜的是，在 Go 语言中，并没有严格意义上的面向对象编程。相反，Go 采用了结构体和面向接口的编程风格。

通过结构体，可以定义具有字段和方法的自定义类型，从而模拟面向对象的数据结构。

而通过接口，可以定义一组方法集合，使得不同类型的对象可以共享相同的行为。

尽管 Go 语言没有传统意义上的类和继承的概念，但它仍然可以通过结构体和接口来实现面向对象的设计思想。通过合理地使用结构体和接口，可以在 Go 中实现高效、灵活且易于维护的代码。



## 结构体
在 Go 中，可以通过`type struct`定义结构体，每一个结构体都可以拥有自己的属性和方法。从而实现类似面向对象编程的风格。

我们先来看一个完整的示例。

```go

    Name string
    Age  string
}

```
