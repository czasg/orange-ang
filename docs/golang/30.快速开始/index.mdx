运行一个 Go 程序通常有两种方法：

1. 通过 `go run` 命令直接运行。
2. 通过 `go build` 命令编译成可执行文件后再运行。

一般的开发环境中，我们通常使用 `go run` 直接运行服务，这样可以在开发过程中快速进行调试和测试。

而在生产环境中，我们则更倾向于通过 `go build` 将源码编译成可执行文件，并将可执行文件部署到生产环境中运行，这样可以提高程序的执行效率和安全性。

当然了，现在生产环境大多都是通过镜像的方式部署，因此这个编译动作一般会在镜像打包的过程中执行。

## 1.第一个Go程序

现在准备我们第一个 **Hello Go 程序**~

:::note 前提准备

1. linux环境
2. go1.20.0+

:::

### 1.1.初始化目录（可选）

首先，我们创建一个 go-learn 的根目录，并在该目录下创建一个 hello 子目录。

```python
>>> mkdir -p /go-learn/hello

>>> cd /go-learn/hello
```

### 1.2.源代码

在 hello 目录下创建一个 main.go 文件，并打开编辑器，写入以下源码：

```python
>>> pwd
/go-learn/hello

>>> vim main.go
...
```

```go title="/go-learn/hello/main.go" showLineNumbers
package main

import "fmt"

func main() {
    fmt.Println("Hello, Golang")
}
```

我们来简单翻译一下上述源码：
- `package main` 表示这个文件属于 main 包。每一个 go 文件都需要指定一个 `package` 包。
- `func main()` 表示程序的入口函数。即程序启动时会从 `main` 开始执行。
- `fmt` 是 go 的标准库之一，提供了格式化 I/O 和字符串处理的功能。在这里，我们使用它来输出 "Hello, Golang" 到控制台。


### 1.3.运行程序

<BrowserWindow>
    <Tabs>
        <TabItem value="go-run" label="go run" default>

            ```go

        </TabItem>
        <TabItem value="go-build" label="go build">

            >>> ./hello

            ```

        </TabItem>
    </Tabs>
</BrowserWindow>

至此，我们的第一个 Hello Go 程序就完成了。
